## 의사소통

- **언어:** 한국어 선호. 기술 용어는 영어 그대로.
- **설명 깊이:** 배경 / 의도 / 직관 / 예시 모두 포함된 상세 설명 선호. 단답형 대답은 피할 것.
- **과장 경계:** "재앙", "심각", "논리적 불일치" 등 과장된 프레이밍 자제. 객관적·담백한 평가 선호.

## 작업 방식

- **Plan-first:** 코드 작성 전 plan/이해 확인 필요. 바로 구현 금지.
- **싸고 명확한 것 우선:** 큰 리팩토링보다 즉시 처리 가능한 작은 개선을 우선 처리. 단, 필요시 큰 리팩토링도 피하지 않음
- **유예 판단:** "실제 필요해질 때까지 유예"를 명시적으로 선호. 미래 대비 오버엔지니어링 경계.
